# Contributing to EnterpriseRAG-AI

## Contribution Guidelines

- Keep pull requests focused and modular
- Avoid unnecessary formatting-only changes
- Write clean and readable code
- Test changes before submitting PRs
- Add documentation wherever necessary

## Pull Request Rules

- One feature/fix per PR
- Link related issue
- Explain implementation clearly
- Avoid low-quality AI-generated code

## Tech Stack

- FastAPI
- Redis
- Kafka
- FAISS
- React
- TypeScript
